## TASK-016 | Sprint 009B — Deployment Blocker Fix (External JS + CSP frame-src)
**Date:** 2026-06-26
**Root Cause 1:** `netlify.toml` CSP `script-src 'self'` blocks all inline `<script>` blocks on Netlify. Tabs and datetime would fail silently on deploy.
**Root Cause 2:** `frame-src` missing `player.vimeo.com` and `player-widget.mixcloud.com`. Sprint 009 music embeds would be silently blocked.
**Fix 1:** Created `js/admin-utils.js` containing datetime IIFE, `s8Tab()` function, and mc-notif-tab handler. Replaced all inline scripts across all 23 admin pages with `<script src="../js/admin-utils.js" defer></script>`. 0 inline scripts remain.
**Fix 2:** Updated `netlify.toml` frame-src to add `https://player.vimeo.com` and `https://player-widget.mixcloud.com`. All prior domains preserved. No `unsafe-inline` added.
**Files Modified:** `js/admin-utils.js` (new), all 23 `admin/*.html`, `netlify.toml`, `documentation/` (4 files)
**Outcome:** Admin dashboard is now Netlify-safe. CSP compliant without `unsafe-inline` for scripts. Media embeds will load correctly on deploy.
